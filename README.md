### 과제 1: 텍스트 분석 도구

한글과 영문 텍스트 파일을 분석하여 다양한 통계 정보를 제공하는 웹 기반 도구

**주요 기능:**
- 총 문자 수 (공백 포함/제외)
- 총 단어 수 (한글: 어절, 영문: 띄어쓰기 기준)
- 총 문장 수 (., !, ? 기준)
- 한글/영문/숫자/특수문자 개수 분석
- 드래그 앤 드롭 파일 업로드
- 반응형 디자인

**기술 스택:**
- HTML5, CSS3, JavaScript
- FileReader API
- 정규표현식

**실행 방법:**
```bash
# 웹 브라우저에서 열기
open task1/index.html
```

### 과제 2: 개인정보 마스킹 도구

텍스트에서 개인정보를 자동으로 탐지하고 안전하게 마스킹 처리하는 보안 도구

**주요 기능:**
- 주민등록번호 탐지 및 마스킹 (123456-1*******)
- 전화번호 마스킹 (010-****-5678)
- 이메일 마스킹 (chu****@gov.kr)
- 주소 마스킹 (서울시 종로구 ****)
- 신용카드 마스킹 (1234-****-****-5678)
- 10가지 테스트 케이스 제공
- 실시간 통계 표시

**기술 스택:**
- Python 3.x (백엔드 스크립트)
- HTML5, CSS3, JavaScript (프론트엔드)
- 정규표현식 (패턴 매칭)
- PEP 8 코딩 스타일

**마스킹 규칙:**

| 개인정보 유형 | 마스킹 규칙 | 예시 |
|------------|-----------|------|
| 주민등록번호 | 뒷자리 전체 | 850315-1******* |
| 전화번호 | 중간 4자리 | 010-****-5678 |
| 이메일 | @ 앞 50% | chu****@gov.kr |
| 주소 | 상세주소 | 서울시 종로구 **** |
| 신용카드 | 중간 8자리 | 1234-****-****-5678 |

**Python 스크립트 실행:**
```bash
cd task2

# 기본 샘플 파일 처리
python mask_personal_info.py

# 특정 파일 처리
python mask_personal_info.py input.txt output.txt

# 결과 확인
cat masked_result.txt
```

**웹 인터페이스 실행:**
```bash
# 웹 브라우저에서 열기
open task2/index.html
```

**테스트 케이스:**
1. 주민등록번호 마스킹 테스트
2. 전화번호 마스킹 테스트
3. 이메일 마스킹 테스트
4. 신용카드 마스킹 테스트
5. 주소 마스킹 테스트
6. 혼합 개인정보 테스트
7. 하이픈 없는 형식 테스트
8. 빈 파일 테스트
9. 개인정보 없는 텍스트 테스트
10. 대량 데이터 테스트

**예외 처리:**
- 파일 없음: "파일을 찾을 수 없습니다" 메시지
- 빈 파일: "빈 파일입니다" 경고
- 개인정보 미발견: "탐지된 개인정보가 없습니다" 메시지
- 권한 오류: "파일 저장 권한이 없습니다" 에러
- 인코딩 오류: UTF-8, CP949, EUC-KR 자동 감지

## 🔒 보안 고려사항

과제 2의 개인정보 마스킹 도구는 다음과 같은 보안 원칙을 따릅니다:

1. **데이터 최소화**: 필요한 부분만 마스킹하여 데이터 유용성 유지
2. **일관성**: 동일한 패턴에 대해 일관된 마스킹 적용
3. **검증**: 정규표현식을 통한 엄격한 패턴 매칭
4. **로깅**: 탐지된 개인정보 통계 제공 (실제 값은 로깅하지 않음)
